# KazLLM Annotator

Локальное веб-приложение для аннотаторов Института языкознания
им. А. Байтурсынова, чтобы создавать единицы данных по инструкции v3
(2026-04-21) для дообучения Qwen3.5 на казахском языке.

**Каждый аннотатор устанавливает программу себе на компьютер.** Данные
хранятся локально в файле `data.db` (SQLite). На сервер ничего не отправляется.

---

## Что умеет программа

1. **Двуязычный интерфейс** — казахский (по умолчанию) и русский, переключатель
   `RU/ҚАЗ` в правом углу шапки. Выбор сохраняется в cookie. Содержимое полей
   аннотатор всегда пишет на казахском (как требует шаблон v3); язык UI и
   сообщений валидации — независимая настройка.
2. **Регистрация и вход.** Имя из аккаунта автоматически идёт в поле
   `Annotator` и в свойство `Author` экспортируемого DOCX
   (требование п. 3.2 и 4.3 инструкции).
3. **Создание единиц через форму** — вместо того чтобы руками править DOCX:
   выпадающие списки для всех закрытых полей (архетип, домен, лицензия,
   причина отвержения для DPO, имя инструмента из Приложения А).
4. **Жёсткая валидация на каждом шаге:**
   - **JSON-схемы для всех 55 инструментов**: автоподстановка пустого
     скелета аргументов и mock_result по кнопке «Заполнить пустым шаблоном».
     Проверка обязательных и необязательных аргументов, типов, enum-значений.
   - **TOOL_RESULT — это JSON, не казахский текст.** Лейбл и валидация
     это явно указывают.
   - **TOOL_RESULT должен совпадать с `expected mock_result`** из предыдущего
     ASSISTANT-тура (требование п. 3.1 шаблона).
   - **Cross-domain warning**: `kgd.*` инструменты не должны вызываться
     в `domain: banking` и т.п.
   - **Соответствие `[TOOLS_IN_SCOPE]` факту**: все вызванные инструменты
     должны быть в scope; все заявленные в scope должны хотя бы раз
     использоваться.
   - **Чередование ролей**: запрет USER→USER подряд, и т.п.
   - **`[TOOLS_IN_SCOPE]` обязателен** для `business_scenario`.
   - **Внутри-аннотаторская дедупликация**: запрет двух единиц с одинаковым
     `unit_code` у одного аннотатора.
   - **Bad alternative ≠ good answer** для DPO (тривиальная пара отбрасывается).
   - Архетипо-зависимые требования (например, `cultural_anchor`
     обязателен для `cultural_reasoning`).
   - Длительность работы 3–45 мин (п. 4.1).
   - ИИН проверяется по контрольной сумме.
   - Источник не может быть пустым или «unknown».
   - **Предупреждения** (не блокируют): рекомендация `thinking_kk` для
     бизнес-сценариев (+13.9 п.п. качества), детектор русского текста
     в казахских полях.
5. **Экспорт в DOCX** ровно по шаблону раздела 3.1, с правильным свойством
   `Author`. Имя файла по шаблону `batch_ГГГГ_ММ_категория_NN.docx`.
6. **Справка простыми словами** — что такое LLM, диалог, тур, инструмент,
   thinking, DPO. Для коллег без IT-фона. Доступна на обоих языках.
7. **Готовые примеры по каждому архетипу.** Кнопка «Примеры» в меню
   открывает страницу с пятью полностью валидными единицами — по одной
   на каждый архетип v3 (`dialogue`, `preference`, `cultural_reasoning`,
   `business_scenario`, `benchmark`). Каждый пример можно одним кликом
   загрузить в форму как шаблон, изучить и адаптировать под свою задачу.
8. **Редактирование сохранённых единиц.** Кнопка «Редактировать»
   в списке открывает форму с полностью загруженными значениями: текст,
   thinking_kk, имя инструмента, аргументы и mock_result. JS разворачивает
   JSON-значения в именованные поля параметров для удобного редактирования.

## Установка (для аннотатора)

### Шаг 1. Установить Python

- **Windows:** скачайте Python 3.11+ с https://www.python.org/downloads/
  При установке поставьте галочку «Add Python to PATH».
- **macOS:** уже установлен, или `brew install python`.
- **Linux:** `sudo apt install python3 python3-venv python3-pip`.

Проверьте: откройте терминал/командную строку и наберите
```
python --version
```
(или `python3 --version`). Должно показать `Python 3.11.x` или новее.

### Шаг 2. Скопировать папку с программой

Сохраните папку `kazllm-annotator` к себе на компьютер, например в
`C:\Users\Имя\Documents\kazllm-annotator` (Windows) или
`~/kazllm-annotator` (macOS/Linux).

### Шаг 3. Установить зависимости

В терминале перейдите в папку с программой и выполните:

```bash
cd kazllm-annotator
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS / Linux:
source .venv/bin/activate

pip install -r requirements.txt
```

### Шаг 4. Запустить

```bash
python app.py
```

Откроется адрес `http://127.0.0.1:5556`. Откройте его в браузере (Chrome,
Firefox, Edge — любой).

При первом запуске зарегистрируйте аккаунт. Полное имя должно совпадать с
тем, как вы будете подписывать DOCX.

Чтобы остановить сервер — нажмите `Ctrl+C` в терминале.

## Что хранится локально

- `data.db` — SQLite с вашими аккаунтами и единицами. Не передавайте этот
  файл другим — он содержит хеши паролей и черновики.
- При экспорте в DOCX файл сохраняется в папку «Загрузки» вашего браузера.

---

## Если что-то сломалось

- **«python: command not found»** — не выполнен Шаг 1, либо в Windows
  не была поставлена галочка «Add Python to PATH».
- **Ошибка при `pip install`** — обновите pip: `python -m pip install --upgrade pip`.
- **Порт 5000 уже занят** — отредактируйте последнюю строку `app.py`,
  поменяйте `port=5000` на `port=5050`.
- **Забыли пароль** — удалите `data.db` и зарегистрируйтесь заново.
  Все ранее введённые единицы будут потеряны, поэтому регулярно делайте
  экспорт DOCX.
